# MarketplaceOrderHistory

## Назначение модели

Модель `MarketplaceOrderHistory` отслеживает историю изменения статусов заказов
с маркетплейсов. Связана с таблицей `marketplace_order_history` в базе данных.

## Атрибуты

### Fillable (массово присваиваемые)

- `marketplace_order_id` - ID заказа с маркетплейса (MarketplaceOrder)
- `marketplace_order_item_id` - ID элемента заказа (MarketplaceOrderItem)
- `status` - статус изменения

### Отсутствуют

- `hidden` - отсутствуют скрытые атрибуты
- `casts` - отсутствуют приведения типов

## Связи с другими моделями

### BelongsTo (многие к одному)

- `item()` - элемент заказа, к которому относится запись в истории
- `order()` - заказ с маркетплейса

## Скоупы и методы

Отсутствуют специфические скоупы или методы.

## Особенности реализации

- Модель для аудита изменений статусов заказов
- Позволяет отслеживать полный жизненный цикл заказа
- Может использоваться для:
    - Анализа времени нахождения заказов в разных статусах
    - Восстановления истории изменений
    - Выявления проблемных этапов обработки заказов
- Хранит связи как с заказом в целом, так и с отдельными элементами заказа
- Не использует трейт HasFactory
