## Общая архитектура

Приложение использует многоуровневую архитектуру моделей:

1. **Управление пользователями** - Ролевая модель с гибкими настройками доступа
2. **Складской учет** - Полный цикл движения материалов от поступления до
   списания
3. **Интеграция с маркетплейсами** - Ozon и Wildberries API
4. **Финансовый модуль** - Начисления, бонусы, транзакции
5. **Справочники** - Централизованное хранение нормативно-справочной информации

## Особенности реализации

- **Soft Deletes** - User, Material, Supplier используют мягкое удаление
- **Relationships** - 60+ связей между моделями (One-to-One, One-to-Many,
  Many-to-Many)
- **Casts & Accessors** - Преобразование данных и вычисляемые атрибуты
- **Observers** - Автоматическая обработка событий моделей
- **Factories** - Генерация тестовых данных

## Ключевые бизнес-процессы

1. **Управление заказами** - От создания до выполнения
2. **Движение материалов** - Поступление → перемещение → расход → списание
3. **Расчет финансов** - Автоматическое начисление зарплат и бонусов
4. **Инвентаризация** - Контроль остатков на складе
5. **Работа с маркетплейсами** - Двусторонняя синхронизация данных
