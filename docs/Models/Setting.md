# Setting

## Назначение модели

Модель `Setting` предоставляет механизм хранения глобальных настроек приложения
в базе данных. Связана с таблицей `settings` в базе данных.

## Атрибуты

### Fillable (массово присваиваемые)

- `name` - название настройки (уникальный ключ)
- `value` - значение настройки

### Отсутствуют

- `hidden` - отсутствуют скрытые атрибуты
- `casts` - отсутствуют приведения типов

## Связи с другими моделями

Отсутствуют явные связи.

## Скоупы и методы

### Статические методы

- `getValue(string $name): ?string` - получает значение настройки по имени
    - Возвращает значение поля `value` для записи с указанным именем
    - Если настройка не найдена, возвращает `null`

## Особенности реализации

- Простая key-value система хранения настроек
- Используется для хранения различных параметров приложения:
    - Настройки бизнес-логики
    - Конфигурационные значения
    - Пороговые значения и лимиты
- Метод `getValue()` предоставляет удобный доступ к настройкам
- Значения хранятся как строки (приведение к нужному типу должно производиться
  на уровне использования)
- Не имеет валидации типов или формата значений
