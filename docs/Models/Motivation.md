# Motivation

## Назначение модели

Модель `Motivation` хранит информацию о мотивационных параметрах для
сотрудников, включая ставки и бонусы. Связана с таблицей `motivations` в базе
данных.

## Атрибуты

### Fillable (массово присваиваемые)

- `user_id` - ID сотрудника (User)
- `from` - дата начала действия мотивационной схемы
- `to` - дата окончания действия мотивационной схемы
- `rate` - базовая ставка
- `bonus` - размер бонуса
- `not_cutter_bonus` - бонус для некроек
- `cutter_bonus` - бонус для кроек

### Отсутствуют

- `hidden` - отсутствуют скрытые атрибуты
- `casts` - отсутствуют приведения типов

## Связи с другими моделями

Отсутствуют явные связи, но неявно связана с моделью `User` через поле
`user_id`.

## Скоупы и методы

Отсутствуют специфические скоупы или методы.

## Особенности реализации

- Хранит временные периоды действия мотивационных схем
- Разделяет бонусы для разных категорий сотрудников (крой и некрой)
- Может использоваться для расчета заработной платы сотрудников
- Поля `from` и `to` позволяют задавать временные интервалы для разных схем
  мотивации
