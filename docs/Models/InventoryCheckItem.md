# InventoryCheckItem

## Назначение модели

Модель `InventoryCheckItem` представляет отдельный товар, проверяемый в ходе
инвентаризации. Связана с таблицей `inventory_check_items` в базе данных.

## Атрибуты

### Fillable (массово присваиваемые)

- `inventory_check_id` - ID инвентаризации (InventoryCheck)
- `marketplace_order_item_id` - ID элемента заказа (MarketplaceOrderItem)
- `expected_shelf_id` - ID ожидаемого стеллажа (Shelf)
- `founded_shelf_id` - ID фактического стеллажа, где товар найден (Shelf)
- `is_found` - флаг, найден ли товар (boolean)
- `is_added_later` - флаг, добавлен ли товар позже в процессе инвентаризации (
  boolean)

### Отсутствуют

- `hidden` - отсутствуют скрытые атрибуты
- `casts` - отсутствуют приведения типов

## Связи с другими моделями

### BelongsTo (многие к одному)

- `expectedShelf()` - стеллаж, где товар должен находиться
- `foundedShelf()` - стеллаж, где товар фактически найден
- `marketplaceOrderItem()` - проверяемый элемент заказа

## Скоупы и методы

Отсутствуют специфические скоупы или методы.

## Особенности реализации

- Закомментирован трейт HasFactory (не используется)
- Является частью сложной системы инвентаризации, которая:
    - Отслеживает расхождения между ожидаемым и фактическим местоположением
      товаров
    - Позволяет добавлять товары в процессе инвентаризации (`is_added_later`)
    - Фиксирует найденные и не найденные товары (`is_found`)
- Связи с двумя стеллажами позволяют:
    - Сравнивать ожидаемое и фактическое размещение
    - Выявлять ошибки хранения
    - Оптимизировать раскладку товаров
- Может использоваться для:
    - Генерации отчетов по инвентаризации
    - Анализа точности хранения
    - Корректировки местоположения товаров
