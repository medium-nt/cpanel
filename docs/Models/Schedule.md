# Schedule

## Назначение модели

Модель `Schedule` хранит информацию о графике работы сотрудников. Связана с
таблицей `schedules` в базе данных.

## Атрибуты

### Fillable (массово присваиваемые)

- `user_id` - ID сотрудника (User)
- `date` - дата работы

### Отсутствуют

- `hidden` - отсутствуют скрытые атрибуты
- `casts` - отсутствуют приведения типов

## Связи с другими моделями

### BelongsTo (многие к одному)

- `user()` - сотрудник, для которого составлено расписание

## Скоупы и методы

Отсутствуют специфические скоупы или методы.

## Особенности реализации

- Простая модель для отслеживания дней работы сотрудников
- Может использоваться для:
    - Планирования смен
    - Расчета рабочего времени
    - Определения доступности сотрудников для распределения заказов
- Не хранит время начала/окончания смены, только дату
- Может иметь несколько записей для одного сотрудника (например, при сменном
  графике)
