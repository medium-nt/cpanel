# InventoryCheck

## Назначение модели

Модель `InventoryCheck` представляет инвентаризацию склада - процесс проверки
наличия товаров на стеллажах. Связана с таблицей `inventory_checks` в базе
данных.

## Атрибуты

### Fillable (массово присваиваемые)

- `status` - статус инвентаризации ('in_progress' | 'closed')
- `comment` - комментарий к инвентаризации
- `finished_at` - дата и время завершения инвентаризации

### Casts (приведения типов)

- `finished_at` - преобразуется в `datetime`
- `status` - приводится к типу `string`

### Свойства (Properties)

- `$status` (string) - текущий статус инвентаризации

## Связи с другими моделями

### HasMany (один ко многим)

- `items()` - элементы проверяемых товаров в рамках данной инвентаризации

## Скоупы и методы

### Accessors (мутаторы/доступы)

- `getCreatedDateAttribute(): ?string` - возвращает дату создания в формате '
  d/m/Y H:i:s'
- `getFinishedDateAttribute(): ?string` - возвращает дату завершения в формате '
  d/m/Y H:i:s'

## Особенности реализации

- Закомментирован трейт HasFactory (не используется)
- Поддерживает два статуса инвентаризации:
    - `in_progress` - в процессе проведения
    - `closed` - завершена
- Хранит дату завершения отдельно от стандартных таймстемпов Laravel
- Предоставляет форматированные даты для отображения в интерфейсе
- Связь с элементами инвентаризации позволяет:
    - Отслеживать проверяемые товары
    - Сравнивать ожидаемое и фактическое местоположение
    - Отмечать найденные и добавленные позже товары
