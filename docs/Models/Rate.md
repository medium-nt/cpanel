# Rate

## Назначение модели

Модель `Rate` хранит информацию о ставках оплаты для сотрудников за работу с
разными материалами. Связана с таблицей `rates` в базе данных.

## Атрибуты

### Fillable (массово присваиваемые)

- `user_id` - ID сотрудника (User)
- `material_id` - ID материала (Material)
- `rate` - базовая ставка оплаты
- `not_cutter_rate` - ставка для сотрудников, не являющихся закройщиками
- `cutter_rate` - ставка для закройщиков

### Отсутствуют

- `hidden` - отсутствуют скрытые атрибуты
- `casts` - отсутствуют приведения типов

## Связи с другими моделями

### BelongsTo (многие к одному)

- `material()` - материал, для которого установлена ставка

## Скоупы и методы

Отсутствуют специфические скоупы или методы.

## Особенности реализации

- Позволяет устанавливать индивидуальные ставки для каждого сотрудника
- Разделяет ставки по типам сотрудников:
    - Базовая ставка (`rate`)
    - Ставка для закройщиков (`cutter_rate`)
    - Ставка для других специалистов (`not_cutter_rate`)
- Гибкая система расчета оплаты:
    - Учитывает сложность материала
    - Разные ставки для разных специалистов
    - Персонализированный подход к каждому сотруднику
- Связь с материалом позволяет:
    - Учитывать стоимость материалов при расчете
    - Дифференцировать оплату в зависимости от типа материала
    - Устанавливать премии за работу с дорогими материалами
- Может использоваться для:
    - Расчета заработной платы
    - Формирования отчетов по оплате труда
    - Анализа производительности и стоимости работ
- Не имеет явной связи с User (только через user_id)
