# Rating Board — Доска рейтинга сотрудников

> Last reviewed: 2026-07-10

## Обзор

Доска рейтинга сотрудников (`/rating_board/{token}/{workshop}`) — это публичная
страница для отображения результатов работы сотрудников в реальном времени.
Доступна без авторизации по токену, показывает лидеров швей, подиум победителей,
статистику по ролям и стикеры FBO/FBS. Данные обновляются через polling каждые
30 секунд.

## Как это работает

### Доступ и авторизация

**Публичный доступ без авторизации:**

- URL: `/rating_board/{token}/{workshop}`
- Токен и workshop проверяются в `RatingBoardController::index()`
- Middleware `auth` НЕ применяется — страница доступна всем по ссылке
- Используется для ТВ-мониторов в цехах

**Polling:**

- JavaScript в `public/rating_board/js/app.js` опрашивает сервер каждые 30
  секунд
- Эндпоинт: `GET /rating_board/data/{token}/{workshop}` возвращает JSON
- JSON содержит: leaders, podium, statistics, timers, winner, stickers_fbo/fbs

### Лидеры — таблица топ швей

**Что показывается:**

- ТОЛЬКО швеи (`role_id=1`) — закройщики и ОТК НЕ попадают в таблицу лидеров
- Метрика: количество заказов, завершённых швеёй сегодня (`completed_at` за
  сегодня)
- Максимум 9 строк в таблице (`RATING_ROLE_IDS = [1]`)
- Сортировка: по убыванию количества заказов DESC

**Константы в сервисе:**

```php
RATING_ROLE_IDS = [1]  // Только швеи для лидеров/подиума
STATISTICS_ROLE_IDS = [1, 4, 5]  // Все 3 роли для статистики
```

### Подиум — победители (top 3)

**Медали:**

- **Gold** (1 место) — швея с максимальным количеством заказов сегодня
- **Silver** (2 место) — вторая по результатам
- **Bronze** (3 место) — третья по результатам
- ТОЛЬКО швеи (`role_id=1`) — закройщики и ОТК не попадают на подиум
- При равенстве заказов — несколько человек на одной позиции (например, 2 gold)

**Фраза золотого медалиста:**

- В подиуме (gold) показывается случайная фраза из массива `GOLD_PHRASES` (100
  вариантов)
- Логика в JS (`app.js`):
    - Человек остался на 1 месте → фраза НЕ меняется (`currentGoldUserId`
      совпадает)
    - Новый человек на 1 месте → новая случайная фраза
    - Старый ушёл и вернулся → НОВАЯ фраза (не сохраняется)
- Примеры фраз: "Высшая лига портновского искусства.", "Мастерство уровня
  год!", "Королева ниток и иголок!"

**Фикс пустого подиума:**

- При пустом подиуме (нет лидеров) раньше ложно срабатывали swap-анимации со
  звуком
- Теперь: `if (!next.gold && !next.silver && !next.bronze) return;`
- Дополнительные проверки `&& g1 && s1` в условиях свапов

### Статистика — все роли, по дням

**Что показывается:**

- ВСЕ 3 роли: швеи (1), закройщики (4), ОТК (5)
- Каждая строка = один день работы одного сотрудника (раньше был агрегат за
  месяц)
- Период: с 1-го числа **текущего месяца** до **вчера** (заказы за сегодня НЕ
  попадают)
- Сортировка: по дате DESC (свежие дни сверху)
- Лимита НЕТ — показываем все записи за месяц

**Метрики по ролям (кастомная дата для каждой):**

| Роль          | Колонка даты           | Описание               |
|---------------|------------------------|------------------------|
| Швея (1)      | `completed_at`         | Дата завершения заказа |
| Закройщик (4) | `cutting_completed_at` | Дата завершения кроя   |
| ОТК (5)       | `packed_at`            | Дата упаковки          |

**Медаль в статистике:**

- Gold получает **только швея** с максимальным количеством заказов за день (
  `dailyMax`)
- Несколько швей с одинаковым максимумом → все получают gold
- Закройщики и ОТК — **всегда без медалей** (`medal = null`)

**Формат значения:**

```
"{d.m.y} выполнено {N} заказов!"
```

Например: "10.07.26 выполнено 15 заказов!" — это реальная дата работы
сотрудника (не текущая).

### Смена в статистике — из индивидуального расписания

**Что изменилось:**

- Раньше: смена в карточке статистики = текущая смена ЦЕХА (`shiftNameAt` по
  `shift_schedule`)
- Теперь: смена = **смена сотрудника в тот день** из таблицы `schedules` (
  индивидуальное расписание)
- Ключ мапы: `"user_id|date"` → читаем `schedules.user_id` + `schedules.date` →
  `schedules.shift_id`
- Связь: `Schedule belongsTo Shift`
- Если в `schedules` нет записи на дату → `shift = ''` (пусто в карточке)

**Отличие от заголовка доски:**

- `shiftNameAt` (по `shift_schedule`) остаётся для ЗАГОЛОВКА доски (текущая
  смена цеха)
- В статистике — индивидуальная смена сотрудника из `schedules`

### Стикеры FBO/FBS

**Что показывается:**

- FBO стикер: количество заказов в статусе 5 («На стикеровку») с FBO-поставкой
- FBS стикер: количество заказов в статусе 5 с FBS-поставкой
- Обновляются через polling (каждые 30 секунд)

**Звук при обновлении:**

- Звук играется только если `newValue > oldValue` (заказы добавились)
- Раньше — при любом изменении (баг)
- Логика в `animateStickerValue` в `app.js`

### Победитель дня

- Показывается в отдельном блоке
- Определяется на основе данных за сегодня
- Формат: "Победитель дня: {имя} — {N} заказов!"

### Таймеры

- Таймер рабочего дня: показывает время с начала смены до конца
- Настройки: `working_day_start` и `working_day_end` (global settings)
- Формат: "Часы работы: 09:00 - 18:00" + обратный отсчёт

### Фронт — очистка от демо-данных

- `resources/views/rating_board/index.blade.php` очищен от захардкоженных
  демо-данных
- Контейнеры (лидеры, подиум, статистика, победитель) пустые при загрузке
- Заполняются при первом polling из JSON
- Заголовок «Статистика за {текущий месяц}» — динамически через `now()->month`

## Ключевые файлы

- `app/Services/RatingBoard/RatingBoardDataService.php` — основная бизнес-логика
    - Константы `RATING_ROLE_IDS = [1]` (лидеры/подиум) и
      `STATISTICS_ROLE_IDS = [1, 4, 5]` (статистика)
    - `getLeaders()` — строит таблицу лидеров (только швеи, лимит 9)
    - `getPodium()` — строит подиум (gold/silver/bronze, только швеи)
    - `getStatistics()` — статистика по дням (3 роли, период до вчера, медаль
      рекордсмену, смена из schedules)
    - `getTimers()` — таймеры рабочего дня
    - `getWinner()` — победитель дня
    - `getStickers()` — стикеры FBO/FBS
    - `buildLeaders()` — ограничение 9 строк (`if ($i >= 9) break;`)
- `app/Http/Controllers/RatingBoardController.php` — контроллер доски
    - `index()` — рендер страницы (проверка токена и workshop)
    - `data()` — JSON-эндпоинт для polling (возвращает все данные)
- `public/rating_board/js/app.js` — фронтенд-логика
    - `GOLD_PHRASES` — массив из 100 случайных фраз для золотого медалиста
    - `applyPodiumDiff()` — анимация подиума с фиксами пустого подиума
    - `animateStickerValue()` — звук стикеров только при росте (
      `newValue > oldValue`)
    - `renderStatistics()` — рендер статистики с медалями (`card.medal`)
    - Polling каждые 30 секунд
- `resources/views/rating_board/index.blade.php` — шаблон страницы
    - Очищен от статичных демо-данных (контейнеры пустые)
    - Заголовок статистики динамический (`now()->month`)
    - Standalone blade с собственным layout
- `tests/Feature/RatingBoardDataServiceTest.php` — тесты сервиса
    - 11 тестов `getStatistics` (период до вчера, группировка по дням, медали)
    - Тесты лидеров — только швеи, закройщики/ОТК не попадают
- `public/rating_board/css/app.css` — стили доски
- `public/rating_board/index.html` — публичный entry point

## Бизнес-правила

- Лидеры и подиум — **ТОЛЬКО швеи** (`role_id=1`), закройщики (4) и ОТК (5) не
  участвуют
- Статистика показывает **все 3 роли** (швеи, закройщики, ОТК) но **по дням**,
  не агрегатом за месяц
- Период статистики: с 1-го числа текущего месяца до **вчера** (заказы за
  сегодня не включаются)
- Медаль gold в статистике получает **только швея** с дневным рекордом (
  `dailyMax`), закройщики и ОТК — всегда без медалей
- Смена в карточке статистики — из **индивидуального расписания** (`schedules`),
  не из `shift_schedule`
- Если в `schedules` нет записи на дату → смена пустая (`shift = ''`)
- Максимум **9 лидеров** в таблице (`RATING_ROLE_IDS = [1]`)
- Фраза золотого медалиста — случайная из 100 вариантов (JS), меняется только
  при смене человека на 1 месте
- Звук стикеров FBO/FBS — только при **росте** количества (
  `newValue > oldValue`), не при любом изменении
- Фронт не содержит захардкоженных данных — всё заполняется через polling из
  JSON
- Доступ к странице — **без авторизации** по токену (для ТВ-мониторов)
- Polling — каждые 30 секунд, не затрагивает авторизацию (middleware `auth` не
  применяется)
