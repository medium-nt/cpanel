# Salary System — Начисления и тарифы

> Last reviewed: 2026-06-16

## Обзор

Система начисления зарплаты построена на Action-модели: оплата за конкретные
действия (пошив, раскрой, переупаковка, стикеровка). Тарифы настраиваются
индивидуально для каждого сотрудника и материала. Дневные лимиты ограничивают
максимальный объём работы. Бонусы проходят 14-дневный холд перед выплатой.

## Как это работает

### Accrual-команды (cron, routes/console.php)

| Время | Команда                | Описание                   |
|-------|------------------------|----------------------------|
| 00:30 | `accrual:salary-daily` | Оклад (type=fixed_daily)   |
| 00:35 | `accrual:sewing`       | Начисление за пошив        |
| 00:40 | `accrual:cutting`      | Начисление за раскрой      |
| 00:45 | `accrual:repacking`    | Начисление за переупаковку |
| 00:50 | `accrual:sticking`     | Начисление за стикеровку   |

Все команды запускаются ночью после завершения рабочей смены.

### ActionAccrualService

Центральный сервис начислений:

- Обрабатывает выполненные за день заказы
- Рассчитывает сумму на основе тарифов и метража
- Создаёт записи в Transaction (зарплата + бонус)

### Тарифы (Tariff и UserTariff)

**Tariff** — тариф на конкретный материал для конкретного сотрудника:

- `rate` — базовая ставка
- `not_cutter_rate` — ставка для не-закройщиков
- `cutter_rate` — ставка для закройщиков

**UserTariff** — индивидуальные тарифы пользователя:

- Связь User → UserTariff → Tariff
- Поддержка разных ставок для разных ролей

### Motivation — бонусная система

**Motivation** — тированные бонусы по объёму:

- `user_id` — сотрудник, для которого действует тариф
- `from` / `to` — диапазон ширины/количества (пороги для начисления бонуса)
- `rate` — базовая ставка бонуса
- `bonus` — сумма бонуса
- `not_cutter_bonus` — бонус для не-закройщиков
- `cutter_bonus` — бонус для закройщиков

Пример: если швея отшила 50-100 метров → бонус X, если 100-200 → бонус Y.

**Бизнес-логика:**

- Бонусные ставки рассчитываются на основе объёма выполненных работ
- Разные ставки для закройщиков (cutter_bonus) и не-закройщиков (
  not_cutter_bonus)
- Система применяется через ActionAccrualService при расчёте дневных начислений
- Каждое начисление бонуса создаёт запись в Transaction с 14-дневным холдом

### Дневные лимиты

`User::dailyLimitReached()`:

1. Считает метраж выполненных и в работе заказов за сегодня
2. Сравнивает с лимитом из настроек (seamstress_daily_limit /
   cutter_daily_limit)
3. Если лимит достигнут → сотрудник не может брать новые заказы

### Холд-система бонусов (14 дней)

`TransactionService::activateHoldBonus()` (ежедневно в 00:20):

1. Находит бонусы с `accrual_for_date` старше 14 дней
2. Меняет статус: 0 (pending/held) → 1 (paid/доступен)
3. После холда бонус доступен к выплате

**Зачем 14 дней:** защита от фрода и возможность отмены при обнаружении брака.

### Интеграция с Transaction

Каждое начисление создаёт запись в Transaction:

- `is_bonus=false` → зарплата (доступна сразу)
- `is_bonus=true` → бонус (14-дневный холд)
- `transaction_type` = 'in' (доход сотрудника)
- `accrual_for_date` — дата за которую начислено

## Ключевые файлы

- `app/Services/ActionAccrualService.php` — центральный сервис начислений
- `app/Console/Commands/` — accrual:* команды (5 штук)
- `app/Models/Tariff.php` — тарифы по материалам
- `app/Models/UserTariff.php` — индивидуальные тарифы
- `app/Models/Motivation.php` — тированные бонусы
- `app/Services/TransactionService.php` — создание транзакций
- `routes/console.php` — расписание начислений

## Бизнес-правила

- Начисления проходят ночью после закрытия смен
- Разные ставки для закройщиков и не-закройщиков (cutter_rate vs
  not_cutter_rate)
- Бонусы удерживаются 14 дней перед выплатой
- Дневные лимиты в метрах ограничивают нагрузку
- Все начисления логируются как Transaction с accrual_for_date
- **Важно:** удаление заказов в статусе "новый" (0) не влияет на начисления,
  так как за них ещё не производилось оплаты
