# Finance — Транзакции и мотивация

> Last reviewed: 2026-06-22

## Обзор

Финансовая система отслеживает все денежные операции: зарплаты, бонусы, штрафы,
расходы компании. Транзакции создаются автоматически при начислениях (зарплата,
пошив, раскрой) и вручную (штрафы, корректировки). Бонусы проходят 14-дневный
холд. Есть мотивационная система с тированными ставками.

## Как это работает

### Transaction — основная модель

**Поля:**

- `user_id` — сотрудник
- `title` — описание операции
- `marketplace_order_item_id` — привязка к заказу (опционально)
- `accrual_for_date` — дата, за которую начислено
- `amount` — сумма
- `status` — 0=pending (бонус на холде), 1=paid (выплачен), 2=company (операция
  компании)
- `transaction_type` — 'in' (доход сотрудника) / 'out' (расход/штраф)
- `is_bonus` — false=зарплата, true=бонус
- `paid_at` — дата выплаты

### TransactionService — создание и управление

**Создание транзакции `store()`:**

1. Валидация данных
2. Проверка баланса компании (защита от овердрафта для company-операций)
3. Создание записи с соответствующим статусом

**Типы операций:**

- **Зарплата** (`is_bonus=false, status=1`): доступна сразу
- **Бонус** (`is_bonus=true, status=0`): холд 14 дней, затем → status=1
- **Штраф** (`transaction_type='out'`): списание за отмену заказа
- **Компания** (`status=2`): внутренние операции

### Холд-система бонусов

`activateHoldBonus()` (ежедневно в 00:20):

1. Находит бонусы с `accrual_for_date` > 14 дней назад
2. Меняет `status`: 0 → 1
3. Бонус становится доступным для выплаты
4. Логирование активации

**Зачем 14 дней:** защита от фрода, возможность отмены при обнаружении брака.

### Аналитика и отчёты

- `getLastPayouts()` — группировка выплат по дате с диапазонами accrual дат
- `getSumOfPayout()` — чистая сумма выплаты за период
- `getOldestUnpaidSalaryEntry()` — самая старая невыплаченная зарплата
- `getTotalByType()` — сумма по типам (бонус/зарплата/компания) с фильтром по
  дате
- `getCashflowFiltered()` — чистый денежный поток по сотруднику и дате выплаты
- `getSeamstressBalance()` — баланс швеи (зарплата vs бонус с учётом холда)

### Штрафы

`penalizeUserForOrderCancellation()`:

- Создаёт транзакцию типа 'out' при отмене заказа
- Привязывается к конкретному `marketplace_order_item_id`
- Отражается в балансе сотрудника

### Stack — очередь заказов швеи

**Stack** — отслеживает параллельную нагрузку:

- `seamstress_id` — швея
- `stack` — текущее количество заказов в работе
- `max` — пиковое количество за смену

**StackService:**

- `incrementStackAndMaxStack()` — добавляет заказ, обновляет peak
- `reduceStack()` — завершает заказ, сбрасывает max если очередь пуста
- `clearAllStacks()` — ежедневный сброс в 00:01

Зачем: для расчёта нагрузки и предотвращения перегрузки швей.

### Motivation — мотивационная система

Тированные бонусы по объёму работы:

- Диапазоны ширины (`from`, `to`) → разная ставка бонуса
- Отдельные ставки для закройщиков (`cutter_bonus`) и не-закройщиков (
  `not_cutter_bonus`)
- Привязка к конкретному сотруднику

## Ключевые файлы

- `app/Services/TransactionService.php` — создание и аналитика транзакций
- `app/Models/Transaction.php` — модель транзакции
- `app/Models/Motivation.php` — мотивационные ставки
- `app/Models/Rate.php` — ставки по материалам
- `app/Models/Stack.php` — очередь заказов
- `app/Services/StackService.php` — управление стеком
- `routes/console.php` — activateHoldBonus (00:20), clearAllStacks (00:01)

## Бизнес-правила

- Зарплата доступна сразу, бонусы — через 14 дней
- Штрафы создаются при отмене заказов сотрудником
- Stack отслеживает текущую нагрузку швеи, сбрасывается ежедневно
- Защита от овердрафта при company-операциях
- Все операции логируются с accrual_for_date для корректной отчётности
- **Важно:** удаление заказов в статусе "новый" (0) не создаёт транзакций,
  так как за них не производилось начислений
