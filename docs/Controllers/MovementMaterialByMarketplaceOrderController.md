# MovementMaterialByMarketplaceOrderController

**Путь:**
`app/Http/Controllers/MovementMaterialByMarketplaceOrderController.php`

## Общее назначение

Контроллер для просмотра расходов материалов на заказы с маркетплейсов.
Реализует только отображение списка заказов.

## Зависимости и сервисы

- `App\Models\Order` - модель заказа
- `Illuminate\Http\Request` - HTTP запросы

## Список методов

### index(Request $request)

- **Описание:** Отображение списка всех расходов материалов на заказы с
  маркетплейсов
- **URL:** `/movements_by_marketplace_order`
- **Метод:** GET
- **Возвращает:** `Illuminate\View\View`
- **Представление:** `movements_by_marketplace_order.index`
- **Данные:**
    - `title` (string) - заголовок страницы
    - `orders` (LengthAwarePaginator) - коллекция заказов с пагинацией (100 на
      страницу)
- **Фильтрация:**
    - `where('type_movement', 3)` - только заказы типа расход материала на
      маркетплейс
    - `latest()` - сортировка по убыванию даты создания
- **Параметры запроса:** `$request` (не используется в текущей реализации)

## Работа с моделями данных

- **Order:** Основная модель для хранения информации о расходах материалов на
  заказы

## Типы движения

- `type_movement = 3` - расход материала на заказ маркетплейса

## Права доступа

- Контроллер не использует Gate или middleware для проверки прав доступа
- Предполагается использование глобальных middleware для аутентификации

## Особенности реализации

- Контроллер реализует только операцию чтения (index)
- Не включает методы для создания, редактирования или удаления расходов
- Большой размер пагинации (100 записей) предполагает просмотр больших списков
- Весь функционал ограничен только отображением существующих данных

## Валидация

- Отсутствует явная валидация
- Контроллер только читает данные из базы
