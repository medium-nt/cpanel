# StoreMovementMaterialFromSupplierRequest

## Назначение класса валидации

Класс `StoreMovementMaterialFromSupplierRequest` предназначен для валидации
запросов на создание перемещения материалов от поставщика. Этот класс
используется для регистрации поступления новых материалов от поставщиков на
склад.

## Правила авторизации (метод authorize)

```php
public function authorize(): bool
{
    return true;
}
```

- Все авторизованные пользователи имеют право выполнять данный запрос
- Дополнительные проверки авторизации не требуются

## Правила валидации (метод rules)

```php
public function rules(): array
{
    return [
        'supplier_id.required|exists:suppliers,id' => 'Поставщик обязателен.',
        'material_id.*.required' => 'Материал обязателен.',
        'material_id.*.exists:movement_materials,id' => 'Материал не найден.',
        'quantity.*.required' => 'Количество обязательно.',
        'quantity.*.min' => 'Количество должно быть больше или равно нулю.',
        'comment' => 'string|min:3|max:255|nullable',
    ];
}
```

### Поля валидации:

- **`supplier_id`** - Идентификатор поставщика
    - Правило определено некорректно - содержит сообщение об ошибке вместо
      правил валидации
    - Корректные правила должны быть: `required|exists:suppliers,id`

- **`material_id.*`** - Массив идентификаторов материалов
    - Правило определено некорректно - содержит сообщение об ошибке вместо
      правил валидации
    - Корректные правила должны быть: `required|exists:movement_materials,id`

- **`quantity.*`** - Массив количеств поступающих материалов
    - Правило определено некорректно - содержит сообщение об ошибке вместо
      правил валидации
    - Корректные правила должны быть: `required|numeric|min:0`

- **`comment`** - Комментарий к поступлению
    - `string` - Должно быть строкой
    - `min:3` - Минимальная длина - 3 символа
    - `max:255` - Максимальная длина - 255 символов
    - `nullable` - Может быть пустым (null)

## Особенности валидации

1. **Некорректная структура правил** - В методе `rules()` правила определены в
   неверном формате:
    - Вместо правил валидации указаны сообщения об ошибках
    - Правильный формат для `supplier_id`:
      `'supplier_id' => 'required|exists:suppliers,id'`

2. **Массивная валидация** - Класс ожидает получение массивов данных для
   массового поступления материалов

3. **Проверка существования** - Проверяется существование поставщика и каждого
   материала в соответствующих таблицах

4. **Допустимость нулевого количества** - Комментарий в правилах говорит о том,
   что количество должно быть больше или равно нулю, что позволяет
   регистрировать нулевые поступления (возможно для документальных целей)

5. **Обязательный комментарий** - Минимальная длина комментария - 3 символа, что
   позволяет добавить краткое описание поступления

6. **Несоответствие сообщений** - В методе `messages()` указано сообщение "
   Количество должно быть больше нуля", что противоречит комментарию в правилах

## Заметки о реализации

В коде класса присутствуют критические ошибки в определении правил валидации,
которые предотвращают корректную работу валидации. Рекомендуется исправить
структуру правил:

```php
// Правильная структура правил
public function rules(): array
{
    return [
        'supplier_id' => 'required|exists:suppliers,id',
        'material_id.*' => 'required|exists:movement_materials,id',
        'quantity.*' => 'required|numeric|min:0',
        'comment' => 'string|min:3|max:255|nullable',
    ];
}
```

После исправления правил валидация будет работать корректно, обеспечивая
проверку существования поставщика, материалов и корректности количественных
данных.
