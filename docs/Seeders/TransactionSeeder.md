# TransactionSeeder

**Файл:** `database/seeders/TransactionSeeder.php`

## Назначение сида

Создание тестовых финансовых транзакций для демонстрации работы бухгалтерии.

## Создаваемые данные

- **Транзакции:** 4 записи
    - 2 приходные операции (продажи)
    - 2 расходные операции (выплата зарплат)

## Особенности генерации

- Создает реальные сценарии финансовых операций
- Показывает разные типы транзакций
- Привязывает расходные операции к конкретным сотрудникам

## Зависимости от других сидов

- `UserSeeder` - требует наличия пользователей с ID 2 (кладовщик) и 3 (швея)

## Выполняемые операции

Создает 4 транзакции:

1. Приход 1000 от продажи №1
2. Приход 500 от продажи №2
3. Расход 300 - зарплата швее (пользователь ID 3)
4. Расход 900 - зарплата кладовщику (пользователь ID 2)

## Тестовые данные

```
Транзакция 1:
- title: 'получение денег за продажу No1'
- amount: 1000
- transaction_type: 'in'
- status: 1 (подтверждена)

Транзакция 2:
- title: 'получение денег за продажу №2'
- amount: 500
- transaction_type: 'in'
- status: 1 (подтверждена)

Транзакция 3:
- user_id: 3 (швея)
- title: 'Выдача зарплаты швее'
- amount: 300
- transaction_type: 'out'
- status: 1 (подтверждена)

Транзакция 4:
- user_id: 2 (кладовщик)
- title: 'Выдача зарплаты кладовщику'
- amount: 900
- transaction_type: 'out'
- status: 1 (подтверждена)
```

## Примечания

Все транзакции имеют статус 1 (подтверждены), что означает их исполнение.
