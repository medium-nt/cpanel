# UserSeeder

**Файл:** `database/seeders/UserSeeder.php`

## Назначение сида

Создание базовых пользователей системы для всех ролей.

## Создаваемые данные

- **Пользователи:** 15 записей
    - 5 предустановленных пользователей для каждой роли
    - 10 дополнительных пользователей через фабрику

## Особенности генерации

- Создает одного пользователя для каждой основной роли
- Использует простые пароли для легкого входа в систему разработки
- Дополняет 10 случайными пользователями через фабрику для реалистичной картины

## Зависимости от других сидов

- `RoleSeeder` - требует наличия всех ролей в системе

## Выполняемые операции

Создает 5 основных пользователей:

1. Администратор (role_id=3): 1@1.ru / 111111
2. Кладовщик (role_id=2): 2@2.ru / 222222
3. Швея (role_id=1): 3@3.ru / 333333
4. Закройщик (role_id=4): 4@4.ru / 444444
5. Сотрудник ОТК (role_id=5): 5@5.ru / 555555

## Тестовые данные

```
Администратор:
- email: 1@1.ru
- password: 111111
- role: admin

Кладовщик:
- email: 2@2.ru
- password: 222222
- role: storekeeper

Швея:
- email: 3@3.ru
- password: 333333
- role: seamstress

Закройщик:
- email: 4@4.ru
- password: 444444
- role: cutter

ОТК:
- email: 5@5.ru
- password: 555555
- role: otk
```

## Примечания

Пароли intentionally простые для удобства разработки. В production-среде следует
использовать более сложные пароли или механизм восстановления паролей.
