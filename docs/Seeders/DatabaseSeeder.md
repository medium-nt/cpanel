# DatabaseSeeder

**Файл:** `database/seeders/DatabaseSeeder.php`

## Назначение сида

Главный сеeder, который управляет порядком выполнения всех остальных сеедеров в
приложении.

## Создаваемые данные

Не создает данные напрямую, а только вызывает другие сеедеры в определенной
последовательности.

## Особенности генерации

- Выполняет сеедеры в строгом порядке, учитывая зависимости между ними
- Обеспечивает целостность данных при первоначальном наполнении базы

## Зависимости от других сидов

Не имеет зависимостей, является корневым сеедером.

## Выполняемые операции

Последовательно вызывает следующие сеедеры:

1. `RoleSeeder` - создание ролей пользователей
2. `UserSeeder` - создание пользователей
3. `TypeMaterialSeeder` - создание типов материалов
4. `MaterialSeeder` - создание материалов
5. `SupplierSeeder` - создание поставщиков
6. `ShelfSeeder` - создание полок склада
7. `OrderSeeder` - создание заказов
8. `MovementMaterialSeeder` - создание движений материалов
9. `MarketplaceItemSeeder` - создание товаров для маркетплейсов
10. `TransactionSeeder` - создание транзакций
11. `ScheduleSeeder` - создание графика работы
12. `SettingsSeeder` - создание настроек системы

## Порядок выполнения

Порядок критически важен для соблюдения целостности данных:

- Сначала создаются справочники (роли, типы материалов, полки)
- Затем пользователи и материалы
- После этого заказы и связанные с ними данные
- В конце настройки и графики
