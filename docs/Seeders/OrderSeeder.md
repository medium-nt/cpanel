# OrderSeeder

**Файл:** `database/seeders/OrderSeeder.php`

## Назначение сида

Создание тестовых заказов для демонстрации работы системы управления заказами.

## Создаваемые данные

- **Заказы:** 2 записи
    - 1 заказ поставки (тип 1, выполнен)
    - 1 заказ производства (тип 2, новый)

## Особенности генерации

- Создает заказы разных типов и статусов
- Привязывает заказы к ответственным сотрудникам
- Устанавливает дату завершения для выполненного заказа

## Зависимости от других сидов

- `UserSeeder` - требует наличия пользователей с ID 2 (кладовщик) и 3 (швея)
- `SupplierSeeder` - требует наличия поставщика с ID 1

## Выполняемые операции

Создает 2 заказа:

1. Поставка от поставщика №1, выполнена кладовщиком №2
2. Заказ на производство, назначен швее №3

## Тестовые данные

```
Заказ 1 (поставка):
- type_movement: 1 (приход)
- status: 3 (выполнен)
- supplier_id: 1
- storekeeper_id: 2
- comment: 'тестовая поставка'
- completed_at: текущая дата

Заказ 2 (производство):
- type_movement: 2 (расход)
- status: 0 (новый)
- supplier_id: 1
- seamstress_id: 3
- comment: 'тестовый заказ'
```

## Примечания

- type_movement: 1 - приход материалов, 2 - расход материалов
- status: 0 - новый, 3 - выполнен
